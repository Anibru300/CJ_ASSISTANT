## 4. Confirmación de PostgreSQL como base de datos

n8n está configurado explícitamente para usar PostgreSQL:

```yaml
DB_TYPE=postgresdb
DB_POSTGRESDB_HOST=postgres
DB_POSTGRESDB_PORT=5432
DB_POSTGRESDB_DATABASE=${POSTGRES_DB:-cj_assistant}
DB_POSTGRESDB_USER=${POSTGRES_USER:-cj_user}
DB_POSTGRESDB_PASSWORD=${POSTGRES_PASSWORD:-cambiar_en_produccion}
```

**Evidencia de conexión exitosa:**

```text
$ docker exec cj_postgres psql -U cj_user -d cj_assistant -c "SELECT COUNT(*) FROM information_schema.tables WHERE table_schema = 'public';"
 count
-------
   109
(1 row)
```

n8n creó **109 tablas** en PostgreSQL, confirmando que no está usando SQLite.

### Conclusión de evaluación

El despliegue quedó **estructuralmente preparado para evolucionar a producción** porque:

- Usa PostgreSQL como base de datos.
- Tiene Redis disponible para escalabilidad futura.
- Persiste datos fuera de los contenedores.
- Separa claramente el rol de n8n como orquestador.

Sin embargo, antes de producción se deben completar:

1. Cambiar todas las credenciales por defecto.
2. Configurar HTTPS y dominio propio.
3. Definir límites de CPU y memoria.
4. Configurar respaldos automáticos.
5. Evaluar el modo external del Python task runner.
