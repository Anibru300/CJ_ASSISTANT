# Core / Policies

## Propósito

Definir reglas transversales de gobierno del sistema. No son reglas de negocio específicas de un dominio, sino políticas que aplica todo CJ_OS.

## Responsabilidades

- Definir políticas de seguridad, prioridad y privacidad.
- Regular cuándo una acción puede automatizarse y cuándo requiere confirmación.
- Establecer límites de costos de APIs externas.
- Definir criterios de escalamiento a humanos.
- Gobernar el uso de IA dentro del sistema.
- Definir políticas de respaldo y recuperación.

## Tipos de políticas

| Política | Descripción |
|----------|-------------|
| Seguridad | Quién puede ver o modificar qué. |
| Prioridad | Qué eventos se atienden primero. |
| Privacidad | Qué datos no pueden salir de la empresa. |
| Automatización | Qué acciones pueden ejecutarse solas. |
| Costos | Límites de uso de APIs de IA. |
| Escalamiento | Cuándo y cómo avisar a un humano. |
| IA | Qué tareas pueden resolverse con IA. |
| Backups | Qué datos se respaldan y con qué frecuencia. |
